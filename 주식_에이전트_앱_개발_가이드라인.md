### 왜 Streamlit + Python인가?

개인용으로 사용하면서 주니어 수준에서 개발하기에는 Python 기반 대시보드가 가장 효율적이다. React/Next.js 같은 풀스택 웹 앱은 프론트엔드·백엔드·배포까지 관리 포인트가 많아 MVP를 만들기까지 시간이 오래 걸린다. 반면 Streamlit은 Python 코드만으로 UI를 구성할 수 있고, pandas/numpy/matplotlib 등 데이터 분석 라이브러리와 자연스럽게 통합된다. 로컬에서 실행하면 서버 비용도 들지 않는다.

    """한국/미국/글로벌 주식 데이터 통합 수집기"""

    def get_price_data(self, ticker: str, market: str, period: str = "1y"):
        """시장에 따라 적절한 API로 시세 데이터 수집"""
        if market == "KR":
            return self._fetch_krx(ticker, period)
        else:  # US, ETF 등
            return self._fetch_yfinance(ticker, period)

        stock = yf.Ticker(ticker)
        hist = stock.history(period=period)
        return hist

        end = datetime.today()
        start = end - timedelta(days=365 if period == "1y" else 180)

            start.strftime("%Y%m%d"),
            end.strftime("%Y%m%d"),

            return {
                "income": stock.financials,
                "balance": stock.balance_sheet,
                "cashflow": stock.cashflow

### Phase 2: 포트폴리오 최적화 엔진 (2~3주)

이 단계에서는 사용자의 현재 포트폴리오를 분석하고, 다양한 최적화 전략으로 최적 비중을 계산한 뒤, 주어진 예산으로 실제 매수 수량을 제안한다.

**3-4. 포트폴리오 최적화 핵심 로직**

```python
# portfolio/optimizer.py
import numpy as np
import pandas as pd
from pypfopt import EfficientFrontier, risk_models, expected_returns
from pypfopt.discrete_allocation import DiscreteAllocation, get_latest_prices

class PortfolioOptimizer:
    def __init__(self, price_data: pd.DataFrame):
        """
        price_data: 각 종목의 일별 종가 DataFrame
                    (columns = 종목코드, index = 날짜)
        """
        self.prices = price_data
        self.mu = expected_returns.mean_historical_return(price_data)
        self.cov = risk_models.sample_cov(price_data)

    def optimize_max_sharpe(self):
        """샤프 비율 최대화 전략"""
        ef = EfficientFrontier(self.mu, self.cov)
        weights = ef.max_sharpe(risk_free_rate=0.04)
        cleaned = ef.clean_weights()
        performance = ef.portfolio_performance(verbose=False)
        return {
            "weights": dict(cleaned),
            "expected_return": performance[0],
            "volatility": performance[1],
            "sharpe_ratio": performance[2]
        }

    def optimize_min_volatility(self):
        """최소 변동성 전략 (보수적 투자자용)"""
        ef = EfficientFrontier(self.mu, self.cov)
        weights = ef.min_volatility()
        cleaned = ef.clean_weights()
        performance = ef.portfolio_performance(verbose=False)
        return {
            "weights": dict(cleaned),
            "expected_return": performance[0],
            "volatility": performance[1],
            "sharpe_ratio": performance[2]
        }

    def calculate_discrete_allocation(self, weights: dict, budget: float):
        """
        실제 예산으로 몇 주를 사야 하는지 계산
        - weights: 최적화된 비중 딕셔너리
        - budget: 투자 가능 예산 (원 또는 달러)
        """
        latest_prices = get_latest_prices(self.prices)
        da = DiscreteAllocation(
            weights, latest_prices, total_portfolio_value=budget
        )
        allocation, leftover = da.greedy_portfolio()
        return {
            "allocation": allocation,    # {"AAPL": 5, "삼성전자": 10, ...}
            "leftover": leftover,        # 잔여 현금
            "invested": budget - leftover
        }
```

**3-5. 다중 시장 통합 처리 시 주의사항**

한국과 미국 주식을 동시에 다룰 때는 환율 변환이 필수적이다. 원화 기준으로 통일하거나 달러 기준으로 통일해야 비중 계산이 정확해진다. 환율 데이터는 yfinance의 `KRW=X` 티커나 한국은행 API에서 가져올 수 있다. 또한 한국 시장과 미국 시장의 거래일이 다르므로, 시계열 데이터를 합칠 때 누락 날짜 처리(forward fill 등)가 필요하다.

**3-7. 뉴스 분석 에이전트 구현 예시**

```python
# agent/news_analyzer.py
import anthropic
import json

class NewsAnalystAgent:
    def __init__(self, api_key: str):
        self.client = anthropic.Anthropic(api_key=api_key)

    def analyze_news(self, news_articles: list[dict], portfolio_tickers: list[str]):
        """
        뉴스 기사들을 분석하여 포트폴리오 관련 인사이트 추출

        news_articles: [{"title": "...", "summary": "...", "source": "...", "date": "..."}]
        portfolio_tickers: ["AAPL", "005930", "QQQ", ...]
        """
        prompt = f"""당신은 전문 금융 뉴스 분석가입니다.

아래 뉴스 기사들을 분석하여 다음을 수행하세요:

1. **감성 분석**: 각 뉴스의 시장 영향을 긍정/중립/부정으로 분류
2. **관련 종목 매핑**: 뉴스가 영향을 미칠 포트폴리오 종목 식별
3. **리스크 이벤트 감지**: 급락/급등 가능성이 있는 이벤트 식별
4. **종합 시장 전망**: 현재 시장 분위기 요약

현재 포트폴리오 종목: {portfolio_tickers}

뉴스 기사 목록:
{json.dumps(news_articles, ensure_ascii=False, indent=2)}

아래 JSON 형식으로 응답하세요:
{{
    "market_sentiment": "bullish|neutral|bearish",
    "sentiment_score": -1.0 ~ 1.0,
    "key_events": [
        {{
            "event": "이벤트 설명",
            "impact": "positive|negative|neutral",
            "affected_tickers": ["AAPL"],
            "severity": "high|medium|low"
        }}
    ],
    "ticker_sentiments": {{
        "AAPL": {{"score": 0.7, "reason": "..."}},
    }},
    "summary": "종합 시장 전망 요약"
}}"""

        response = self.client.messages.create(
            model="claude-sonnet-4-6",
            max_tokens=2000,
            messages=[{"role": "user", "content": prompt}]
        )

        return json.loads(response.content[0].text)
```

**3-8. 포트폴리오 매니저 에이전트**

```python
# agent/market_analyst.py

class PortfolioManagerAgent:
    def __init__(self, api_key: str):
        self.client = anthropic.Anthropic(api_key=api_key)

    def generate_recommendation(
        self,
        current_portfolio: dict,
        optimization_result: dict,
        news_analysis: dict,
        macro_data: dict,
        budget: float
    ):
        """
        모든 분석 결과를 종합하여 최종 투자 추천 생성

        current_portfolio: 현재 보유 종목 및 수량
        optimization_result: 수학적 최적화 결과
        news_analysis: 뉴스 분석 에이전트 결과
        macro_data: 경제지표 데이터
        budget: 추가 투자 가능 예산
        """
        prompt = f"""당신은 개인 투자자를 위한 포트폴리오 매니저입니다.

아래 정보를 종합 분석하여 투자 추천 리포트를 작성하세요.

## 현재 포트폴리오
{json.dumps(current_portfolio, ensure_ascii=False, indent=2)}

## 수학적 최적화 결과 (Mean-Variance 기반)
{json.dumps(optimization_result, ensure_ascii=False, indent=2)}

## 뉴스/시장 감성 분석
{json.dumps(news_analysis, ensure_ascii=False, indent=2)}

## 거시경제 지표
{json.dumps(macro_data, ensure_ascii=False, indent=2)}

## 추가 투자 예산
{budget:,.0f}원

다음 형식으로 리포트를 작성하세요:

1. **시장 환경 요약**: 현재 거시 환경과 시장 전망 (2~3문장)
2. **포트폴리오 진단**: 현재 포트폴리오의 강점/약점
3. **최적화 추천**: 수학적 최적화 결과에 뉴스/시장 상황을 반영한 조정 의견
4. **구체적 매수 가이드**: 예산 {budget:,.0f}원으로 어떤 종목을 몇 주 매수할지
5. **리스크 요인**: 주의해야 할 리스크와 대응 방안
6. **신뢰도 평가**: 이 추천의 신뢰 수준 (high/medium/low)과 그 이유

⚠️ 중요: 이 추천은 참고용이며, 최종 투자 결정은 사용자 본인이 해야 합니다.
투자에는 원금 손실 위험이 있습니다."""

        response = self.client.messages.create(
            model="claude-sonnet-4-6",
            max_tokens=3000,
            messages=[{"role": "user", "content": prompt}]
        )

        return response.content[0].text
```

## 5. 핵심 알고리즘 설명

### 5-1. Mean-Variance Optimization (평균-분산 최적화)

현대 포트폴리오 이론(MPT)의 핵심이다. 각 종목의 기대수익률과 공분산 행렬을 기반으로, 주어진 위험 수준에서 기대수익률을 최대화하는 (또는 주어진 기대수익률에서 위험을 최소화하는) 비중을 찾는다. PyPortfolioOpt 라이브러리가 이 계산을 간단하게 처리해 준다.

### 5-2. Black-Litterman 모델

순수한 수학적 최적화의 한계를 보완하는 모델이다. 시장 균형 비중(시가총액 기준)을 시작점으로, 투자자의 주관적 전망(예: "삼성전자가 향후 6개월간 15% 상승할 것이다")을 결합하여 비중을 조정한다. LLM 에이전트의 뉴스 분석 결과를 이 모델의 "뷰(view)"로 변환하면 시너지가 높아진다.

### 5-3. Discrete Allocation (이산 배분)

최적화 결과는 "AAPL 23.5%, 삼성전자 18.2%..."처럼 연속적인 비중이지만, 실제로는 정수 단위의 주식만 매수할 수 있다. DiscreteAllocation 알고리즘은 예산 제약 하에서 최적 비중에 가장 가까운 정수 주식 수를 계산한다.

## 6. 개발 시 주의사항

### 법적/윤리적 주의사항

이 앱은 **개인 참고용 도구**로만 사용해야 한다. LLM이 생성한 투자 추천은 절대 금융 자문(financial advice)이 아니다. 앱 내에 "본 정보는 투자 권유가 아니며, 투자 결정의 책임은 사용자에게 있습니다"라는 면책 문구를 반드시 포함해야 한다. 또한 이 앱을 타인에게 유료로 제공하면 자본시장법상 "투자자문업" 등록 의무가 발생할 수 있으므로 주의해야 한다.

### 기술적 주의사항

- **API 레이트 리밋**: yfinance, pykrx 등 무료 API는 과도한 호출 시 차단될 수 있으므로 캐싱을 적극 활용한다.
- **데이터 지연**: 실시간 데이터가 아닌 15~20분 지연 데이터임을 인지해야 한다.
- **환율 리스크**: 한국/미국 종목 혼합 시 환율 변동이 수익률에 영향을 미친다.
- **LLM 비용 관리**: Claude API 호출당 비용이 발생하므로, 불필요한 반복 호출을 피하고 결과를 캐싱한다.
- **백테스팅 필수**: 최적화 전략을 실제 자금으로 사용하기 전에 과거 데이터로 반드시 성능을 검증한다.

## 7. 확장 아이디어

MVP를 완성한 후 고려할 수 있는 확장 기능들이다.

- **자동 리밸런싱 알림**: 포트폴리오 비중이 목표 대비 일정 이상 벗어나면 알림 전송 (이메일/텔레그램)
- **백테스팅 모듈**: 최적화 전략의 과거 성과를 시뮬레이션
- **감성 점수 시계열**: 뉴스 감성 점수의 변화 추이를 차트로 표시
- **섹터/테마 분석**: 섹터 로테이션 전략 지원
- **RAG 기반 심층 분석**: 벡터 DB에 뉴스와 리서치 리포트를 저장하고, LLM이 검색하여 분석
- **증권사 API 연동**: 한국투자증권 등의 API로 실제 주문까지 자동화 (고급)
